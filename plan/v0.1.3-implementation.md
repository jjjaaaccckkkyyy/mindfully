# v0.1.3 Implementation Plan: Agent System

## Overview

Build the core agent system combining:
- **LangGraph.js** for graph-based orchestration
- **OpenClaw patterns** for agent loop (minimal tools, layered prompts, lane concurrency)

## Phase 1: Database Schema

### Tables

| Table | Purpose |
|-------|---------|
| `agents` | Agent configurations |
| `executions` | Agent run history |
| `agent_tools` | Tool assignments to agents |
| `memories` | Persistent memories (user + agent scoped) |

### Files to create:

**`server/src/db/migrations/XXX_create_agents_tables.sql`**
- `agents` table with: id, name, description, model, tools (jsonb), memory_enabled, system_prompt, max_tokens, temperature, user_id, created_at, updated_at
- `executions` table with: id, agent_id, input, output (jsonb), status, error, token_usage (jsonb), started_at, completed_at
- `agent_tools` table with: id, agent_id, tool_id, enabled
- `memories` table with: id, user_id, agent_id, content, embedding (vector), created_at, updated_at

## Phase 2: Core Agent System (Core Package)

### 2.1 Tools System

**`core/src/tools/index.ts`**
- Export: `ToolRegistry`, `createTool`, `getTools`

**`core/src/tools/registry.ts`**
- `ToolRegistry` class
- `register(tool: Tool)`, `get(name: string)`, `list()`
- Built-in tools: Read, Write, Edit, Bash

**`core/src/tools/builtin/index.ts`**
- `ReadTool` - Read file contents
- `WriteTool` - Write file contents  
- `EditTool` - Edit file sections
- `BashTool` - Execute shell commands
- Each implements `Tool` interface

**`core/src/tools/mcp/client.ts`**
- MCP client for connecting to MCP servers
- `MCPClient` class
- `connect(serverUrl)`, `listTools()`, `callTool(name, args)`

**`core/src/tools/types.ts`** (already exists)
- `Tool`, `ToolResult`, `ToolSchema` interfaces

### 2.2 Memory System

**`core/src/memory/index.ts`**
- Export: `MemoryService`, `createMemoryService`

**`core/src/memory/service.ts`**
- `MemoryService` class
- `add(entry: MemoryEntry)`, `search(query, options)`, `getRecent(agentId, limit)`
- Hybrid: Markdown storage + Qdrant vector search

**`core/src/memory/qdrant.ts`**
- Qdrant client wrapper
- `QdrantClient` class
- `upsert(collection, points)`, `search(collection, query, limit)`
- `createCollection(collection, dimension)`

**`core/src/memory/markdown.ts`**
- Markdown file storage
- `MarkdownStore` class
- `save(path, content)`, `load(path)`, `append(path, content)`
- Stores in: `~/.mindful/memories/{userId}/{agentId}/`

**`core/src/memory/types.ts`** (already exists)
- `MemoryEntry`, `MemoryQuery`, `MemoryResult` interfaces

### 2.3 9-Layer Prompt Builder

**`core/src/agents/prompt/builder.ts`**
- `PromptBuilder` class
- `build(options: PromptBuildOptions): string`
- 9 layers:
  1. Core Instructions
  2. Tool Definitions (JSON Schema)
  3. Skills Registry
  4. Model Aliases
  5. Protocol Specs
  6. Runtime Info
  7. Workspace Files (IDENTITY.md, AGENTS.md, MEMORY.md)
  8. Bootstrap Hooks
  9. Inbound Context

**`core/src/agents/prompt/layers/`**
- `core.ts` - Layer 1: Core instructions
- `tools.ts` - Layer 2: Tool definitions
- `skills.ts` - Layer 3: Skills registry
- `model.ts` - Layer 4: Model aliases
- `protocol.ts` - Layer 5: Protocol specs
- `runtime.ts` - Layer 6: Runtime info
- `workspace.ts` - Layer 7: Workspace files
- `hooks.ts` - Layer 8: Bootstrap hooks
- `context.ts` - Layer 9: Inbound context

### 2.4 Skills System

**`core/src/agents/skills/registry.ts`**
- `SkillsRegistry` class
- Auto-discover from: `~/development/mindful/skills/`
- `load()`, `get(name)`, `list()`, `match(query)`

**`core/src/agents/skills/types.ts`**
- `Skill` interface
- `SkillDefinition`, `SkillTool` types

### 2.5 Lane Queue (Concurrency)

**`core/src/agents/runtime/lanes.ts`**
- `LaneQueue` class
- Lanes:
  - `main` - User conversations (serial per session)
  - `cron` - Scheduled jobs (parallel)
  - `subagent` - Spawned sub-agents (parallel)
- `enqueue(lane, task)`, `dequeue(lane)`, `run(lane, task)`

### 2.6 Agent Runner (LangGraph)

**`core/src/agents/runner.ts`**
- `AgentRunner` class
- Uses `@langchain/langgraph` StateGraph
- Flow: input → prompt → LLM → tool call → execute → repeat → output

**`core/src/agents/runtime/graph.ts`**
- LangGraph state definition
- Nodes: `agent`, `tools`
- Edges: conditional (continue/stop)

**`core/src/agents/runtime/executor.ts`**
- `AgentExecutor` class
- `run(input, options)`, `stream(input, options)`
- Handles: state management, tool execution, event streaming

## Phase 3: Server API (tRPC Routers)

### 3.1 Agent Router

**`server/src/router/agent.ts`**
```typescript
agentRouter = router({
  list: publicProcedure.query(...),     // List user's agents
  byId: publicProcedure.query(...),    // Get single agent
  create: protectedProcedure.mutation(...),  // Create agent
  update: protectedProcedure.mutation(...),  // Update agent
  delete: protectedProcedure.mutation(...),  // Delete agent
})
```

### 3.2 Execution Router

**`server/src/router/execution.ts`**
```typescript
executionRouter = router({
  run: protectedProcedure.mutation(...),     // Run agent (async)
  stream: protectedProcedure.mutation(...),  // Stream output
  status: protectedProcedure.query(...),     // Check status
  history: protectedProcedure.query(...),    // Execution history
  cancel: protectedProcedure.mutation(...), // Cancel running
})
```

### 3.3 Tool Router

**`server/src/router/tool.ts`**
```typescript
toolRouter = router({
  list: publicProcedure.query(...),      // List available tools
  register: protectedProcedure.mutation(...), // Register custom tool
  byId: publicProcedure.query(...),       // Get tool details
})
```

### 3.4 Memory Router

**`server/src/router/memory.ts`**
```typescript
memoryRouter = router({
  search: protectedProcedure.query(...),    // Search memories
  add: protectedProcedure.mutation(...),    // Add memory
  list: protectedProcedure.query(...),      // List memories
  delete: protectedProcedure.mutation(...), // Delete memory
})
```
